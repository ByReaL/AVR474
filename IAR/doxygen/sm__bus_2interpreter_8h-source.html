00020 /*
00021 Things to watch for:
00022   > Ttimeout = 25mS-35mS (clk held low)
00023   > Tbuf = buss free time after stop > 5uS.
00024   > Tpor = 500mS max = startup time allowed before device must respond to SMBus cmds
00025   > Tpdn = power-down time =  >=2.5secs of bus low  (3.1.4.2)
00026 
00027 Ttimeout (Bus Timeout) is detected by reasserting a wdog timer
00028 each time a transition occurs in the TWI state machine.
00029 
00030 Tbuf is assured by requiring that the user exit the state machine
00031 completely after sending or receiving a STOP condition, rather than
00032 looping around internal to the state machine to send the next START.
00033 
00034 Tpor is dependent on the list of startup tasks.  The Power-On Reset itself
00035 is determined to have occurred via the interrupt.
00036 
00037 Tpdn is determined by:
00038 
00039 
00040 SMBus requires that the TWI module act both as a Slave device and, on occasion,
00041 as a Master.  The management of the hardware for operating between these two
00042 modes is handled in the TWI.C module.  Essentially, the TWI module is kept in
00043 Slave mode by default.  When in IDLE state *AND* the bus is Idle, it checks for
00044 the presence of a command in the Master Command buffer.  If present, it will
00045 initiate the switch to Master Mode and commence the transmission.
00046 
00047 Note that to know for certain that the bus is Idle, we must rely on TWSR.
00048 Fortunately, TWSR does in fact accurately reflect the condition of the bus
00049 regardless of whether the Mega406 is actually communicating on it or not.
00050 
00051 */

00159 Status Handler:
00160 
00161 Current     TWSR                                            Next
00162 State       Status  Meaning         Actions                 State
00163 ----------  ------  ------------    ----------------------  ------------
00164 Idle        0x60    saw SLA w/WR    set TWEA                Wait4Cmd
00165 
00166             0xA8    saw SLA w/RD    goto ReplyData          n/a
00167 
00168 Wait4Cmd    0x80    got CMD         store Cmd Byte,     
00169                                     set TWEA                Wait4RW
00170         
00171 Wait4RW     0x80    got data byte   goto Wait4Data handler      n/a
00172 
00173             0xA0    SLA incoming    set up data for Reply       Idle
00174                                     (based on Cmd rcvd) 
00175 
00176 Wait4Data   0x80    got data        store Data byte,
00177                                     set TWEA                Wait4Data
00178 
00179             0xA0    got Stop        flag MsgDone.           Idle
00180             
00181 ReplyData   0xA8    must send first   load TWDR, and set      ReplyData
00182                                       byte of Reply       TWEA only if not last
00183 
00184             0xB8    got ACK; send   Load TWDR, and set    ReplyData
00185                                     out more data.        TWEA only if not last
00186 
00187             0xC0    got NAK         if have more, flag Error;   Idle
00188                                     if no more, done.           Idle
00189
